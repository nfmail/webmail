# Internationalization

NF Mail uses [Lingui](https://lingui.dev) with gettext `.po` catalogs.
English is the source language; Hungarian is the maintained translation.

## Architecture

- **Catalogs**: `locales/<locale>/messages.po` (edit these), compiled to
  `locales/<locale>/messages.ts` by `npm run i18n:compile` (committed; CI
  fails if compilation would change them — `npm run check:i18n`).
- **Explicit IDs**: message IDs are dotted keys (`calendar.title`). There is
  no macro/SWC plugin — the runtime resolves IDs from the compiled catalogs,
  so the bundler needs no i18n transform.
- **Fallback**: entries with an empty `msgstr` fall back to the English text
  at compile time (`fallbackLocales` in `lingui.config.ts`). A missing ID
  renders the ID itself.
- **Runtime layer** (`i18n/`):
  - `i18n/client.tsx` — `useTranslations(namespace?)`, `useLocale`,
    `useFormatter`, `useMessages` hooks for client components
  - `i18n/server.ts` — `getTranslations`, `getLocale` for server components
    and route handlers
  - `i18n/navigation.ts` — locale-aware `Link` / `useRouter` / `usePathname`
  - `i18n/routing.ts` — the locale registry, default locale, and
    `NEXT_PUBLIC_LOCALE_PREFIX` mode (`never` | `always` | `as-needed`)
  - `proxy.ts` — negotiates the request locale (NEXT_LOCALE cookie →
    Accept-Language → default) and rewrites/redirects into the `[locale]`
    segment

## Adding or changing strings

1. Add the ID + English text to `locales/en/messages.po`, and the Hungarian
   translation to `locales/hu/messages.po` (empty `msgstr` = falls back to
   English).
2. Run `npm run i18n:compile` and commit the updated `messages.ts` files.
3. Use it in code: `const t = useTranslations('my_ns'); t('my_key')`.

Guards: `npm run test:translations` checks en/hu ID parity and that every
static `t("...")` call site resolves to an existing ID.

## Adding a locale

Add the locale to `lingui.config.ts` and `i18n/routing.ts`, create
`locales/<locale>/messages.po` covering the en ID set, add the native name to
the language switcher, compile, and extend the guard suite expectations.
