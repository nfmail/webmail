# Internationalization

NF Mail uses [Lingui](https://lingui.dev) with gettext `.po` catalogs.
English is the source language; Hungarian is the maintained translation.

## Architecture

- **Catalogs**: `locales/<locale>/messages.po` (edit these), compiled to
  `locales/<locale>/messages.ts` by `npm run i18n:compile` (committed; CI
  fails if compilation would change them — `npm run check:i18n`).
- **Natural-text IDs**: the message ID is the English source text itself
  (`t("Save")`, `t("Hello {name}", { name })` — classic gettext). There is no
  macro/SWC plugin — the runtime resolves IDs from the compiled catalogs, so
  the bundler needs no i18n transform. A missing translation renders the ID,
  i.e. the English text: fallback is inherent.
- **Retained dotted keys**: dynamic template-literal call sites
  (`t(\`sidebar.mailboxes.${role}\`)`) resolve enum-like families that stay
  dotted; they are marked "retained" in the catalogs (25 families).
- **Compile-time fallback**: entries with an empty `msgstr` fall back to the
  English text at compile time (`fallbackLocales` in `lingui.config.ts`).
- **Runtime layer** (`i18n/`):
  - `i18n/client.tsx` — `useTranslations(namespace?)`, `useLocale`,
    `useFormatter`, `useMessages` hooks for client components
  - `i18n/server.ts` — `getTranslations`, `getLocale` for server components
    and route handlers
  - `i18n/navigation.ts` — locale-aware `Link` / `useRouter` / `usePathname`
  - `i18n/routing.ts` — the locale registry, default locale, and
    `NEXT_PUBLIC_LOCALE_PREFIX` mode (`never` | `always` | `as-needed`)
  - `proxy.ts` — negotiates the request locale (NEXT_LOCALE cookie →
    Accept-Language → default) and rewrites/redirects into the `[locale]`
    segment

## Adding or changing strings

1. Write the English text directly in code:
   `const t = useTranslations(); t("Do the thing")` (ICU works inline:
   `t("{count, plural, one {# item} other {# items}}", { count })`).
2. Add the same text as `msgid` to both `.po` catalogs; put the Hungarian
   translation in `locales/hu/messages.po` (empty `msgstr` = English shown).
3. Run `npm run i18n:compile` and commit the updated `messages.ts` files.

The settings search index is a generated static module
(`components/settings/settings-search-content.ts`); extend it when a settings
surface gains new searchable entries.

Guards: `npm run test:translations` checks en/hu ID parity and that every
static `t("...")` call site resolves to an existing ID.

## Adding a locale

Add the locale to `lingui.config.ts` and `i18n/routing.ts`, create
`locales/<locale>/messages.po` covering the en ID set, add the native name to
the language switcher, compile, and extend the guard suite expectations.
